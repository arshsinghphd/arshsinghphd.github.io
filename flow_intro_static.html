      In this set of wikis, I describe an approach to the classical problem of optimizing the flow from a source to a sink through a Flow Network. I have written the solutions in Python. I will describe and explain parts of the code and show how to put them together.

      Introduction

      Let's start by clearing some terms: a definition of a flow network follows soon, but for an intuition, visualize a network of an assortment of pipes connected via multi-sized couplings. There is only one inlet, say a pump, and one outlet, maybe a large container, a sump. We are given the capacity of each pipe (based on its diameter, thickness of the walls, material of construction etc.). The task is to set the pump such that we flow the maximum amount of liquid through the system, no more no less. If we try to flow more than max. possible, we will disrupt the system: causing leakages at joints, burst pipes etc.

      It may seem like a simple problem at the first glance, but is not. Later in this wiki, playing around with a toy example, I will show how, if we use brute force this seemingly simple problem has exponential time-complexity of at least O[2m], where m is the number of pipes/edges in the network. Even then, we may not converge to the correct solution.

      If you are not familiar with time complexity, for now you can understand it as follows: as the number of edges increase from 2 to 8 (4 times), the number of operations needed to solve the problem go from 4 to 256 (64 times). If the edges increase to 32, the brute force solution will not be able to finish this search on 512 MB RAM, running into memory issues.

      This problem has variations for many seemingly unrelated real world problems. One of these is circulation in a network, or the minimum flow problem which is used in assembling genome from small fragments of DNAs.

      While there are many more elegant solutions to the Maximum flow problem, closer to how a human would visually look for a path, it is important to see how the computer sees this problem. A brute force solution or a naive solution demonstrates just that - trying to solve this problem without using human like intuition or heuristics.

      A Flow Network

      A classical Flow Network is a directed graph G with V, a set of vertices or couplings, and E is given set of edges between these vertices. A Flow Network has the following characteristics:

      1. One source. In a flow network there should be only one source. If there are more than one sources they can be connected to one source using edges of infinite capacity and be reduced to a flow network.
      2. One sink. In a flow network there should be only one source. If there are more than one sinks they can be connected to one source using edges of infinite capacity and be reduced to a flow network.
      3. Edges are a subset of all possible connections between vertices, say u and v in V such that u != v. That is to say, it is not necessary for each vertex to be directly connected to every other vertex.
      4. Each edge has a capacity, the maximum flow it can carry. We require this capacity to be a positive integer (int: ce > 0). We can flow any non-negative flow in any edge (0 <= fe <= ce).
      5. There cannot be self-directed edges or reverse edges. i.e. no (u, u) edges in the network graph; and if (u, v) is in the graph, we cannot have (v, u). These are to avoid infinite loops in the search algorithm.

      Example of a Flow Network (Cormen, et al.)

      The following is a visual depiction of a network with six vertices and nine edges. Name of the vertex is inside the circle depicting it, and the capacity is marked on the edge. The direction of the edges are shown by an arrowhead.

      1 is the source and 6 is the sink. Note that a source has no incoming edges, and a sink has no emerging edges, which is how these are defined.

      Brute Force Search And Time Complexity

      We can see that max flow out of 1 can be 29 (16 + 13) and max flow into 6 can be 24 (20 + 4). The upper limit to the max flow in this network is 24. But can we flow 24 across the network? In fact, the max flow through this network is 23. See if you can get to a max flow paths before reading on about the brute force search description below.

      A brute force search for paths that start in 1 and end in 6 requires choosing among the possible ~29 permutations of the edges.

      The possible paths between 1 and 6, and the max flow in these nine paths are shown in the table below:

      Max FlowPath
      1121, 2, 4, 6
      2041, 2, 4, 3, 5, 6
      3041, 2, 4, 3, 5, 4, 6
      4041, 3, 5, 6
      5041, 3, 2, 4, 6
      6071, 3, 5, 4, 6
      7041, 3, 2, 4, 3, 5, 6
      8041, 3, 2, 4, 3, 5, 4, 6
      9041, 3, 5, 4, 3, 2, 4, 6

      As time consuming as it is, choosing only one of these paths is not the correct answer to the maximum flow problem. We need to search some more.

      In the table you will notice that I have assigned a max flow to each path, while each edge in the path may have different caps. The flow in any path is limited to the capacity of the edge with lowest capacity on the path. These edges with the lowest capacity are called critical edges. Other than the critical edges, rest of the edges are not used to their full capacity.

      Since the non-critical edges may be able to carry more flow, we may be able to find a parallel path going from source to sink through these non-critical edges. The brute force search for these parallel paths that allow for the highest flow-through requires sorting through the 29-1 possible combinations, to see which can co-exist and have the max flow. The time complexity of this search is O[2p].

      The brute force search time to converge on the max possible flow in flow network is O[2m+2p]. Where, m is the no. of edges and p is the no. of unique paths from source to sink.

      A much more elegant approach to the solution is the Ford-Fulkerson Method. It improves over the brute force method by using more efficient ways to search for the parallel paths, much closer to how a human may find these by visual inspection.
